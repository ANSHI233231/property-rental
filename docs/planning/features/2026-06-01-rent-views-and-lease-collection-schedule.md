# Rent views (Lease-wise / Property-wise tabs) + Lease collection schedule

**Date:** 2026-06-01 · **Scope:** prototype only (`admin/rent.html`, `admin/lease-detail.html`, `assets/styles.css`) · **Type:** view-only enhancement (no new actions).

## A — Admin Rent page: two view tabs

`admin/rent.html` gains a **segmented control** above the table: **Lease-wise** (default) and **Property-wise**. Both views are **cumulative by default** and re-scope to a single month via the shared **Month** filter. View-only (no record/edit affordances — already removed).

- **Shared period control:** the Month select moves up next to the tabs (always visible). Default `All months (cumulative)`; selecting a month re-scopes the active view.
- **Lease-wise tab** = the existing behaviour. Cumulative → one row per lease (`Billed · Collected · Outstanding · Standing`, tiles All/Cleared/Due/Advance). Month → one row per lease for that month (`Rent · Paid · Status`, tiles All/Paid/Partial/Overdue/Pre-paid). The Tenant/Property/Unit/Room/PM filter card + Property→Unit→Room cascade stay in this tab.
- **Property-wise tab** = aggregate the roster **by property**. Always standing columns: `# · Property · Units · Billed · Collected · Outstanding · Standing`; tiles All/Cleared/Due/Advance. Cumulative sums all months; month sums that month. ~4 rows, so the lease-tab filter card is **hidden** here (only the Month control applies).
- **Tab switch** resets the lease-tab filters (search + property/unit/room/pm, via a `change` dispatch on the property select so the cascade rebuilds) so stale `data-unit`/etc. filters don't hide property rows. Month persists across tabs.
- KPI cards are unchanged (org-wide totals, driven by Month only) — same in both tabs.

Built on the existing `render(scope)` → generalised to `render(tab, scope)`; one `ROSTER × MONTHS` dataset still drives table **and** KPIs so totals reconcile. `paginate()` re-reads `tbody.rows`, so a `setFilter(...,'all',...)` after each re-render recomputes pagination + tile counts.

## B — Lease detail: month-by-month collection schedule

`admin/lease-detail.html` gains a **Rent Collection** card (after Rent Change Schedule, before Early Termination; `data-status-show="active,expired,terminated"`). A table listing **every month from lease start → lease end**, generated by JS from the lease's start/end + rent-change schedule:

- Columns: **Month · Rent · Collected · Late Fee · Status**.
- Rent honours the rent-change schedule (₹17,000 → ₹18,000 from 01/05/2026 → ₹19,500 from 01/04/2027).
- Status relative to "today" (2026-06): past months **Received** (collected = rent; a late month shows its **Late Fee** in the separate column), current month **Due** (pending), future months **Upcoming** (collected ₹0).
- A footer summary row: total billed / collected / late fees to date.
